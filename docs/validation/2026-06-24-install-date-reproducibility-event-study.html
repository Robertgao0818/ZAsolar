从「逐检测复现」到「Event-Study 测量设计」

为什么 install-date 无法完全复现、换 ViT backbone 到底买到什么、以及当最终目的是地理聚合 event-study 时该担心什么
日期 2026-06-24   范围 install-date 端到端复现性 + event-study 测量设计   状态 思路梳理（非最终方法学）
一句话 逐检测日期不可能完全复现，因为这条链最底层的原语——单次 LLM 调用——本身是随机变量，而自适应搜索把它放大而非平均。换确定性 ViT 能买到「路径一致性」，但对一个聚合到地理层级的 event-study，一致性既不是瓶颈也不是识别威胁：真正决定结论的是「日期 vs 真值」的系统性偏差和影像 cadence，这两样换 backbone 都不解决，确定化反而把前者藏得更深。

1.测试拿到了什么

3 次完整生产链从零重跑（L0 → L1 → L_census 2023-narrowing → merge_three_layers --with-census → flatten），1:1 复现生产操作、同款模型路由，nothing frozen。脚手架零-API 自检通过、拼接逐字节验证。

2.为什么不可能「完全一致」——四层非确定性

完全一致复现不是工程没做到位，是这条链结构上就做不到。四层噪声从下往上叠，且它们是「相乘 / 分叉」不是「相加」：

不可约Layer 1 · 根因
单次 LLM 调用是随机变量
即使 temp=0，f(同图,同prompt) 也不保证同答案：GPU 浮点非结合性、服务端 batch 组成、MoE 路由、网关无 seed。§3 的 0.81 测的就是这层——啥都冻住，打分器还跟自己 ~19% 不一致。
放大器Layer 2 · 架构
自适应搜索把抖动放大成分叉
下一年抓哪张图依赖上一轮答案 → 边界附近翻一次，整条轨迹送进另一分支（混沌敏感）。指标又是 change-point / argmin（无→有那一帧），极值量天生比均值高方差，翻一帧推动整整一个影像年份。abstain 随机化是最大单一分叉源：同帧时而模糊时而判定，把轨迹随机踢进 undated 吸收态。
输入漂移Layer 3 · 影像
抓图集本身也在变
GEHI/Wayback 可得 vintage、phantom date、--allow-nearest 每次不完全一样。连喂给打分器的图都不是固定的。
概念Layer 4 · 没有真值
过程定义的是分布，不是点
rep1/2/3 是同一分布的 iid 抽样；生产是这个分布里一个偏高覆盖的尾部抽样。要求「完全一致」= 要求一个本质随机的估计量给出可复现的点估计——数学上不成立。你能复现的是它的分布（年份 TVD 0.04 就是证据）。
为什么 rep↔rep 0.74 > vs生产 0.60，而投票救不了生产 你甚至不用换模型——把 Gemini 响应缓存回放，rep↔rep 立刻是 1.0。现在不是 1.0，恰恰因为它在如实采样随机性。生产多定的 16% 不是「更对」，是这一抽恰好少撞几个 abstain 吸收态（survivorship）。majority-of-3 收敛到过程分布的中心，而生产不在中心 → 投票把你拉离生产。
⟂ 与 §3 正好相反：§3 里图冻住，投票去向「固定输入上的 scorer 众数」，生产那次也采同一输入 → 投票拉向生产。同一个「投票」动作，在 §3 帮你匹配生产，在端到端帮你匹配过程——靶心根本不是同一个点。值得在论文点明。

3.两个必须分清的概念

复现-by-replay　vs　复现-by-resample

  • Replay（缓存每个 (图,prompt)→答案 再回放）：能 bit-identical，但只是把某一次抽样钉死，关于稳健性什么都没说。
  • Resample（你做的：nothing frozen 真重跑）：看到方差，因为它测的是估计量本身的随机性。你选了 resample——这对稳健性论文是正确的，0.74/0.60 不是失败，是 replay 永远看不见的真实方差谱。

可约方差　vs　不可约方差

  • 可约（确定化/投票能救）：打分器自抖、随机 abstain。
  • 不可约（任何模型都救不了）：真实安装日落在两个稀疏影像 vintage 之间——从根上不可识别，由影像时间稀疏性设了一个地板。
核心陷阱确定化把「可约方差」消成 0，但真实不确定性活在像素里（一张糊的 2016 历史瓦片上那块小板到底有没有）——换 backbone 不改变像素信息量。它做的是：把「这帧是抛硬币」从「两次给不同答案（你看得见）」变成「两次给同一个稳定但可能错的答案（你看不见）」。方差没消失，它变成了偏差；而 3-rep 散布是你目前唯一能看见不可约不确定性的窗口，确定化会把它焊死。

4.换 ViT / DINOv3-SAT backbone：买到什么、买不到什么

纯就「路径一致性」：能，可能从 rep↔rep 0.74 拉到接近 1.0。机制：本地分类器可做确定性（固定 seed、deterministic algos、argmax 固定 head）→ Layer 1 塌向 1.0；固定 abstain 带（如 0.4<p<0.6）让弃权确定化，吸收态入口不再随机分叉——这是你直觉对的部分，且收益最大。再缓存影像（Layer 3）→ 整条链趋近确定函数。

但代价与盲点：

  • 转方差为偏差（见 §3 陷阱）：一致性是廉价的，缓存 Gemini 也能得 1.0；它对「对不对」零信息。
  • 领域硬证据：solar_zerov2 Phase 0 里 frozen DINOv3-SAT + M2F 即使 sweep 后输给 V3-C floor，头号嫌疑是 GSD 域差（0.6m→6.7cm）。这里影像更糟——多 vintage 历史图、GSD 跨年漂、恰是要靠 abstain 的老糊帧。frozen 几乎一定标定差 → 要么 over-abstain（覆盖崩）要么自信判错。要能用得跨 vintage fine-tune，需带 vintage 标签的 present/absent 对（真实标注成本）。
  • 阈值变成新方差旋钮：abstain 带是手设的，直接决定 undated 率 = 覆盖率 = 年份分布。给定阈值可复现，但头条数字现在敏感依赖一个任意标定选择 → 需独立的阈值灵敏度分析。run-to-run 方差换成了阈值敏感性。
  • 单帧丢比较信号：变化检测的信号本质是成对的（帧 t vs t+1）。单帧 present/absent 弱——项目里 Gemini FP-review 也是多尺度双图才过 bar。
这个想法真正好的版本（steelman）别框成「ViT 替 Gemini 做判断」，框成「帧 t vs t+1 变没变」是一个窄、定义清晰、ViT 该比 LLM 强且能确定化的任务——遥感 change-detection（siamese ViT）是成熟范式，成对输入保留比较信号。✅ 值得做：pairwise-CD + 跨 vintage fine-tune + 验证 differential 标定。❌ 不值得：frozen DINOv3-SAT 单帧 present/absent。

5.真正的目的是 event-study —— 这把问题翻过来了

目标：把 install-date 聚合到不同地理层级，做 event-study，看大停电（load-shedding）/ 大规模 PV 安装的影响。一旦想清楚，分析单元根本不是逐装机日期，而是「地理格 × 时间窗内的新增装机数」。换 backbone 的 ROI 排序随之翻转。

最关键的一句：event-study 清晰度由「日期 vs 真值」决定，复现性碰不到它 event-study 的核心产出是 event-time 动态曲线。随机时序噪声会把「事件后陡升」抹成「缓坡」——在 event-time 上摊平、衰减。而这个抹平来自日期 vs 真值的误差，不是 run-to-run 抖动。确定化消的是 run-to-run 抖动，消不掉 date-vs-truth——那块由影像 cadence + 标定决定，确定性模型照样有。所以：你可以拿到完美 rep↔rep 1.0，同时 event-study 被抹得稀烂。两件事脱钩。

真正威胁识别（identification）的两件事，换 backbone 都不解决

威胁为什么致命backbone swap 的作用
① 与处理相关的系统性时序偏差停电最狠的区往往更密/更穷/板更小/影像更糊 → 装机被系统性定晚 → 「停电→装机」响应被人为推后，测到的是测量伪迹不是行为。误差与处理变量相关 = 有偏，不只是噪声。零帮助；确定化反而把它藏得更深（干净、可复现、但有偏的图）。
② undated 非随机缺失~16% 未定档。若 undated-ness 与地理相关（糊影像/小板区 abstain 多），直接丢 = 选择偏差。改变 undated 的集合，不让缺失随机化；固定阈值可能更空间系统化。
这两件都是 date-vs-truth / missingness 层面，3-rep 测试本身看不见，backbone 也修不了。随机噪声（若经典）只摊平 event-time 曲线；系统性、与处理相关的误差才偏移系数——这才是会沉船的那个。
cadence = 你的时间分辨率硬上限 停电是 2022–2023 逐步加剧的多年现象。关键可行性问题：影像 vintage 间隔能否分辨你要识别的处理变异？若 vintage 年级，你只能说「2022 拍 与 2023 拍 之间装的」，无法放进「Stage 6 那次之前/之后」。任何模型都不改变这个上限。好消息：停电是多年 staggered 的强度变量，年级 bin + 按区强度的 staggered event-study / DiD 多半够用——前提是承认 cadence，别假装有月级日期。

6.行动优先级（backbone swap 掉到第 5）

#动作理由
1估计量改成 interval-censored，别用点日期。每个装机「出现在 vintage t_a 与 t_{a+1} 之间」，用离散时间 hazard / interval-censored survival。拥抱不可约的 cadence 限制，而不是把它藏进假精确的点日期。
2Aggregate-first 回归。因变量直接是「(地理格 × 影像窗) 新增装机数」，不是逐装机日期。已知稳的量（TVD 0.04）直接当 Y；DeepSolar 那一课搬到计量。
3在真实聚合分辨率上测 cell 级复现性。把 3 reps 重算成 ward×year / municipality×year 计数，看 rep↔rep。全国 0.04 稳 ≠ 小 cell 稳。先测这个数再决定换不换 backbone。
4建带 vintage 标签的金标，查 differential bias（不是查复现性）。检验：定档误差是否与板大小/地区/影像质量相关、小板是否聚在处理区。这才是能沉船的检验，且 3-rep 完全看不见。同时用来分离可约/不可约方差。
4b把 undated 当非随机缺失建模。按 cell 报 undated 率，检验是否与处理强度相关；相关则不能直接丢（interval-censored 天然容纳）。防选择偏差。
5（可选，最后）确定性 ViT——且只上 pairwise-CD、跨 vintage fine-tune、验证过 differential 标定的版本，定位为「对 cell 计数的经典噪声做方差缩减」。你所有问题里最轻的一个；仅当第 3 步真的太吵才做。
一句话回到原问题 换 ViT 能不能获得更好的路径一致性？能。但对一个聚合到地理层级的 event-study，路径一致性既不是瓶颈也不是识别威胁所在。瓶颈是影像 cadence 和 date-vs-truth 的系统性偏差——前者任何模型都不动，后者要靠金标查 differential bias 来防，确定化反而会把它藏得更深。把预算从「换 backbone 追一致性」挪到「interval-censored 估计量 + cell 级复现实测 + differential-bias 金标 + undated 缺失建模」，对这篇 paper 的回报高一个量级。

7.两个待定参数（确定后能把设计收得更紧）

(a) 处理强度和地理层级各定在什么分辨率？ward / 自治市 / 省 ？   年 / 季 ？
(b) 影像 vintage 在研究期内典型间隔多久？决定 §5 那个 cadence 可行性判断往哪边落
