执行摘要

一个普查数字的可信度，取决于它对"产生它的人为选择与噪声"有多稳健。本文以应用经济学家审计估计量的方式审计 ZAsolar 屋顶光伏存量—— 对每一个 headline 数字追问：什么会让它动、能动多少。三个问题，三个回答：

稳健性问题防范什么结论
1. 规范稳健性 —— headline 是否调参产物？ 研究者自由度 / oracle 泄漏 操作点只在与所有报告套件不相交的校准网格上拟合，带 ≤1pp 迁移验收门。门失败处（生产 per-detection 链），报告回退到预先声明、fail-closed 的阈值，而非被扫出来的最优值。失败被记录，不被隐藏。
2. 计算可复现性 —— 同输入复现同数字？ 运行间静默漂移 不是模型权重的种子级 bit 复现，我们也不这样声称。保证的是：单一指标内核、config-match 输出复用、冻结的评估 GT、锁定的 merge-mode + 操作点——任何数字都可从源 tile 与 model run 重新推导并追溯。
3. 仪器可靠性 —— 唯一的随机器件让结果动多少？ install-date 层的 LLM 采样噪声 检测器与后处理确定，唯一随机步是 Gemini 打分器。test-retest（642 装机、3 次同输入重跑）：个体 install-date 在生产 k=1 下约 81% 可复现；已确认属于生产模型（McNemar p≈0.65），且非松解码产物（greedy temperature:0 已用且无助于降噪）。可行杠杆=majority-of-3 投票（K=6 认证 → ~0.95 库存加权，3× 成本）。误差近零均值 → 聚合后 install-year 直方图复现到每年 ~1.7%（TVD 0.025）。
一句话诚实版 个体估计比看上去更吵；聚合量比个体估计更稳；唯一"重跑还可能让数字动"的地方——CT per-detection 操作点——被明确标为 fail-closed-pending，而非声称已认证。

全文贯穿两个语义锚点：

  • (i) Cape Town 的评估 GT 是 SAM2 sub-array（Axis-A "A2"）标注，非 installation 级 gold，所以 per-polygon F1 降为诊断，选型走面积聚合离散度（σ_Bw、RMSE）。
  • (ii) 目标是存量级（面积/计数）判断，不是 per-polygon 匹配。

1规范稳健性 —— 校准不是一个被调出来的常数

威胁。在报告套件上扫一个阈值、再用那个被扫出来的数字在同一套件上报告，就是 oracle 泄漏——估计量在自夸。这里并非假想：在 2026-06-07 的评估面上，模型排名随扫描口径翻转——在某一固定阈值下 v3c 赢；在各模型各自最优阈值下 unified_A 赢。一个在评估集上调出来的阈值，因此不是可辩护的操作点。

控制手段：带泄漏检查的锁。操作点只在与所有报告套件不相交的校准网格集上拟合，再做迁移验证（scripts/analysis/lock_operating_point.py）。三条保证由机制强制，而非靠约定：

  • 校准网格不得与任何已登记报告套件相交——相交时脚本抛 [LEAKAGE]（lock_operating_point.py:283–290）。
  • 验收 = 锁定点的 agg_area_F1 在每一个验证套件上都落在 oracle 扫描最大值的 1pp 以内，否则锁失败（ACCEPT_GAP = 0.01，门在 :325–327）。
  • 验证套件只读——只用于检查迁移，绝不用于拟合（operating_point_calibration.yaml 规则 4）。

每次拟合都记录其校准名册、排名规则、逐套件迁移 gap、一个 all_suites_pass 布尔值，以及该次运行的 git_head。

诚实的结果——这道门是有牙齿的。它失败过，而且失败就在生产链上：

ct_aerial_2025_v3c_perdetper-detection5.22 pp未通过验收

即便把整条校准链以 per-detection 模式重跑以消除 merge-mode 混淆，per-detection 锁仍失败——gap 停在 5.22pp。诊断是结构性的：在 per-detection 链上，排名规则（σ_Bw + RMSE）偏好高阈值，而验收指标（agg_F1）峰值更低（t ≈ 0.925），因为 per-detection 预测没有可裁剪的 over-paint，两个指标因而分叉。处置：fail-closed。生产 per-detection 报告使用一个来自 2026-06-07 权衡分析的预先声明阈值，绝不使用在报告套件上扫出来的值。能把这个预声明阈值替换成认证阈值的 unified_A × CT 锁，已用泄漏干净的校准集登记（53 块 Li-KML 网格，距所有训练/报告网格 ≥23.5 km），但尚未拟合——其 tile 待下载。这个缺口是本节承重的 caveat，明说而非粉饰。

两个审计者不可混为一谈的杠杆。

  1. Polygon-confidence 是被校准的；流水线的像素级 post_conf 是固定的。被锁定的杠杆是消费时在 predictions_metric.gpkg 上施加的 per-polygon (confidence, area_m²) 裁剪（polygon_conf_sweep.py:54–57），面积感知，使主导的假阳原型（小、低置信团块）能比大装机砍得更狠。它与 finalizer 里写死的粗固定 post_conf_threshold = 0.85（core/postproc.py:1064–1066）不同。混淆二者会错置校准归属。普查刻意让检测器保持宽松——把 polygon-conf 抬到 σ_Bw 最优的 0.97 会损失约 15pp 计数召回（见校准附录）——并把假阳剔除交给域内的 solar_cls 分类器，而非置信度旋钮。
  2. Merge-mode 是一等的、模型相关的杠杆。v4_canonical.json（跨实验可比性锚）携带七个后处理键、无 merge_mode 字段——它在 2026-06-12 架构优化落地中被剥离，调用方现在必须显式传 --merge-mode；finalize.py 在 CLI 与 JSON 不一致时抛错。这很重要，因为同一模型在两种模式下的离散度差异显著（在 wave-1 面上 unified_A 为 σ_Bw 0.270 / RMSE 395 m² per-detection，对 0.361 / 588 m² pixel-or），所以 pixel-or 与 per-detection 是各自独立、各自拟合的锁。数字总是在标明的 merge-mode 下报告，绝不取跨模式最大值当 headline。

2计算可复现性 —— 可追溯，而非种子级 bit 复现

不声称什么。流水线不提供模型输出的种子级逐 bit 复现。把这点说清楚，是契约的一部分。

保证什么：任何被报告的数字都能通过四件锁定工件，从其输入重新推导并回溯：

  • 单一指标内核。"模型–GT 一致性"的可执行定义是一个纯、无副作用的函数 summarize()（core/area_metrics.py:84）——于 2026-06-12 抽取为"逐 bit 相同逻辑"，使每个调用方算出相同统计量。它消费逐网格的集合论并集面积（A=pred，B=GT，A∩B），产出 Tier-1 套件：agg_area_F1、bulk_pred_gt_ratio、std_ratio_Bw（σ_Bw，主部署裁判的 B 加权离散度）、std_logratio、rmse_m2、thru0_slope、ols_r2。全流程唯一带种子的组件是 bootstrap-CI（seed=0）——它管的是置信区间，不是模型输出。（注：cov50 是 CT 普查覆盖率，来自另一个脚本，非本内核产出，不归入 Tier-1 套件。）
  • Config-match 输出复用。推理仅当 results/<grid>/config.json 与当前代码+参数匹配时才复用既有结果；覆盖需 --force。一个数字不可能静默来自设置不同的陈旧运行。
  • 冻结的评估 GT。JHB CBD 面上的选型锁定到冻结的 clean_gt；把评估 GT 漂到 Li GT 或 micro-T1 被禁止。比较面不随被比较的模型移动。
  • 锁定的 merge-mode + 操作点（§1），外加逐网格溯源字段（imagery_layer_id、model_run_id），把每个数字系到具体源 tile 与具体推理批次。

所以这里的"可复现"指的是在锁定配置下追溯-并-重推，而非确定性重放。这是诚实且有用的论断：第二个分析员，给定相同 config 与冻结 GT，重算出相同 summary——并能指向任一单元背后的确切 tile 与运行。

3仪器可靠性 —— 对唯一的随机步进行测量

检测器（Mask R-CNN）、后处理与面积指标，在权重固定下全是确定的。install-date 层加入一个随机器件：一台 Gemini 视觉模型，逐装机读取一串时间序列航拍 chip，对每个日期判定 PV 是否存在——产出一个存在模式及派生的 install-date。这是全流水线唯一一个在相同输入上重跑可能给出不同答案的地方。注意这不是松温度产物：打分器已请求 greedy temperature:0 解码（gemini_solar_image_review.py:404）；残余方差来自这是一个思考型模型、其内部推理轨迹被采样（无固定种子）。本节测量它让结果动多少，以及什么能消除它。

设计 —— 相同输入上的 test-retest

从 15,859-anchor 的 JHB 存量中分层抽样 244 anchor / 642 装机，刻意超采歧义层（non-monotonic、no-recent-anchor、gemini-failed）——LLM 噪声最可能咬人的地方；headline 随后按存量真实层占比重加权。每个装机的 chip 渲染一次（固定 8 日期窗口，2018–2025）并冻结，然后在这些逐 bit 相同的图像上打分三次。可靠性 = rep-to-rep 一致率；输入恒定，故这隔离的是模型自身的采样方差。chip 与打分调用都为本实验新生成——只有 anchor 列表及其生产状态层被复用（用于分层与存量重加权）；无任何生产 install-date 被回收。（solar_backdating/scripts/validation/llm_reliability_{sample,analyze}.py；原始+汇总在 results/analysis/llm_consistency/reliability/。）

headline 那次跑用了 env 默认的 gemini-3-flash-agent。但生产几乎全部日期由朴素 gemini-3-flash 别名产出（round 类型 initial/bisection/tail/walk_back 加 2023-census 步），gemini-3-flash-agent 只用于狭窄的 anchor_recovery round。为确认这个数字属于生产仪器而非 agent 别名，我们在同一批冻结 chip 上对 150-装机分层切片 ×3 用 gemini-3-flash 重打分——见下"模型一致性"。

模型一致性 —— 这个数字属于生产模型，不是 agent 别名

在 150-装机冻结 chip 切片上用生产 gemini-3-flash ×3 重打分，install-date 复现 0.807，与 agent 别名在相同目标上的 0.820 统计上无法区分（配对 McNemar χ² = 0.20，p ≈ 0.65；150 中 9 对 11 不一致）。在主导 clean 层上二者同样接近（flash 0.925 对 agent 0.85，n=40），且库存加权切片值在 flash 下如有差异也略高（~0.87）。headline 因此迁移到真正产出存量的模型——既不被别名抬高，也不被压低。（results/analysis/llm_consistency/reliability/parity/parity_compare.json。）

~18% 分歧的解剖 —— 它比单步翻转更宽，且无法被显式旋钮过滤

在 117 个分歧装机中：

  • 41%（48）是 present-vs-absent 翻转，非定日错误：在一处微弱装机上，模型在"2018–2025 从不可见"与"在日期 X 出现"之间翻转。这是可见性阈值处的检测裕度抖动。
  • 33%（39）恰好移一个影像步（~12 个月——headline 中位数）：占 69 个仅日期分歧者的 57%。
  • 26%（30）跳两年或以上；这些多步跳跃集中在被标记的小歧义尾。

粗化到 install-year 在这里毫无帮助，而且必须说明为什么：8 日期窗口按年分布（8 个日期落在 8 个不同年份），故 install-year 一致按构造等于精确日期一致——已验证，非假设。年级报告因此在个体上不比精确日期更可靠。两个诊断确认噪声是结构性而非可门控的：它集中在已被标 non-monotonic/uncertain 的约 8% 装机中（重跑翻转率 0.35 对 0.17，对干净单调序列），但置信度无法筛除——642 个目标中 637 个自报置信度 ≥ 0.9，分歧者均值置信度（0.933）仅略低于一致者（0.958）。模型在即将翻转时仍自信，所以置信度裁剪不是那个杠杆。

杠杆 1 —— 投票，别信单次采样（已认证）

生产对每个装机只打分一次（k=1），恰是上面测到的最坏情形。三次重跑里，98.4% 装机已存在明确多数（≥2/3）；只有 1.6%（642 中 10）是不可救的三方分裂。为认证去噪后答案的复现性（不仅是"多数是否存在"），我们把生产模型（gemini-3-flash）重跑扩到冻结切片上的 K=6，在全部十个不相交的 3∣3 划分上比较两次独立的 majority-of-3 抽取。结果：单次抽取复现 0.910（库存加权，两次独立抽取一致），而 majority-of-3 投票复现 0.951（主导 clean 层 0.980）——以 3× 调用换 +4.1pp 提升。majority-of-k 去噪器因此不仅合理而是已测量：它把有效 install-date 复现性抬入 mid-90s。（完全成立的 majority-of-5 需 10 次重跑；6 次代理已对 92% 装机在 ≥4/6 次里钉住众数答案。…/reliability/parity/majority_k.json。）

杠杆 2 —— 钉死解码器：测过了，它不是那个修复

自然假设是方差为松温度产物。它不是。原生打分路径已无条件发送 greedy temperature:0（gemini_solar_image_review.py:404），所以两次可靠性运行本就是 greedy。而 gemini-3-flash——一个直接、非 agent 别名——尽管那个 greedy 请求，仍一样地变（上文 0.807 对 0.820）。残余 rep-to-rep 移动是一个思考型视觉模型在边际可见性日期判定上对推理轨迹采样的内禀属性，不是一个被遗漏未钉的解码器。这关掉了便宜的"config 修复"叙事：诚实的杠杆是投票（杠杆 1）与聚合（下），不是温度旗标。

经济交付物不是任一单个 install-date；它是 install-year cohort 分布（每年出现多少装机）。因为逐目标误差近零均值，它们在聚合下抵消。三次重跑之间：

端到端可复现性 —— 从 0 重跑（执行于 2026-06-23，3 次重跑）

上面的 test-retest 刻意冻结了输入：chip 在单一固定全局窗口上渲染一次再重打分，故它隔离打分器自身采样方差，并且——如 §4 所记——是真实再估计会移动多少的下界。互补的上界问题正是审计者真会问的：若第二个分析员把整条回溯流水线在相同装机上从 0 重跑一遍，多大比例会恢复同一个 install interval？本协议规定那次运行。

设计 —— 重放生产链，不冻结任何东西。由同一份分层样本驱动（§3 的 244 anchor / 642 装机，从 sample_anchors.csv 逐字复用，故两实验直接可比、存量重加权延续）。对每个装机，完整生产回溯链端到端重执行，与存量产出方式 1:1——无任何输入被固定：

  • 影像从 GEHI 重抓（非复用），故 provider 侧 capture 选择 / --allow-nearest 吸附重跑；
  • 自适应日期搜索逐 anchor 重跑（run_adaptive_scan.py）——每次运行逐轮挑自己的探测 vintage，而非 §3 用的单一冻结全局窗口；
  • chip 从新抓的 vintage 重渲染；
  • 打分用生产模型路由——gemini-3-flash 跑 initial/bisection/tail/walk_back 与 2023-census 步，gemini-3-flash-agent 只跑 anchor_recovery，与产出存量的那次运行一致；
  • install-interval 后处理完整重跑（infer_install_dates.py，含孤立 dip 修复 repair_isolated_dips）——故比较落在后处理后的交付物，而非原始分数。

度量 —— install interval，对生产。一致单位是 infer_install_dates.py 发出的 install interval 本身（[install_interval_start, install_interval_end] 括号；point-dated 的 done_appears 装机是退化的单步 ~1 年区间；带界层 installed-during-census、already-present 携其开/闭括号）。两种切法，镜像 §3：

  • 1 次运行 vs 生产——单次从 0 重跑的区间，逐装机对生产存量实际交付的区间比较。这是端到端 k=1 复现性：诚实的"一次独立重跑能否移动这个装机的日期"数字。
  • 3 次运行——互相 + 多数。三次独立从 0 重跑给出三方一致率与一个 majority-of-3 去噪区间，是 §3 认证多数杠杆的端到端类比。

结果（执行于 2026-06-23；3 次从 0 重跑）。相比 §3 的下降很大，也是 headline 发现：自适应再搜索——而非打分器自身采样——才是 install-date 方差的主导来源，且冻结输入测试结构性地看不见它。§3 把每个 anchor 的 chip 冻结在一个全局窗口上仅重打分（k=1 0.81）；端到端每次都重新决定抓哪些 vintage 去打分，一次在关键轮次上的歧义/abstain 判定就能让整个区间脱轨。（已验证为真实信号、非 harness 产物：缓存 chip 存在且可读；重跑确实把生产读得干净的帧重新打成歧义。）

三个读数，按重要性排序：

  1. 被发表的经济量是稳健的。 个体日期会抖，但 install-year 分布几乎不动：rep-to-rep 直方图 TVD 0.037–0.063（对 §3 冻结的 0.025）。误差在相邻年份间是替代性的、非系统的，故在聚合中大体抵消。经济分析消费的直方图能复现；单个装机的括号常常不能。
  2. 逐装机复现性中等，且正确的分母是 rep↔rep，不是 rep↔生产。 两次独立重跑在精确区间上一致 ~0.74（库存加权）；对生产存量的一致更低（~0.60），因为生产是更高覆盖的抽样——它定档 434/642，对重跑紧密聚集的 370/385/371（多 ~16%）。majority-of-3 不抬升 vs 生产（0.58），因为多数去噪收敛到过程中心，而非生产；这与 §3 相反——那里生产就是中心，多数爬到 0.95。
  3. 路由是 churn 机制。 46% 装机（1926 个装机-rep 对中 881 个）被与生产不同的流水线层定档——而且是双向的（如生产-gehi_main→重跑-undated，以及生产-undated→重跑-gehi_main），是真实搜索不稳的签名，而非单向缺陷。2023-census 层是最大单一贡献者：在重跑中变回 no_recent 的 anchor 失去 census 收窄所需的干净 absent→present 范例对。
对论文 —— 诚实的写法 把 cohort-直方图 TVD 与 rep↔rep 自洽作为 headline 复现性证据；把精确逐装机区间（~0.74 库存加权 rep↔rep，~0.60 vs 生产存量）作为诊断地板；并把 ~16% 覆盖差（生产比新鲜重跑定档更多）作为对"生产存量是有利抽样"的诚实 caveat 披露。 不要把"完全一致"当目标，也不要拿生产当真值——后者的分母不是真值，是同分布里的高覆盖抽样；而"完全一致"在这个估计量上不是可达目标，因为冻结测试乐观且结构性盲于自适应搜索这个主方差源——这正是本测试存在的意义。

数字与逐装机审计轨迹在 ~/zasolar_data/geid_temporal/llm_endtoend_20260623/analysis_3rep/ （endtoend_summary.json、derived_cuts.json、endtoend_per_installation.csv）。

成本（每次重跑，实测）：~38 分钟挂钟、~1,300 次 Gemini 调用（L0 601 + L1 651 + census 54）、517 次 anchor GEHI 抓取、~3GB chip——3 次重跑 ≈ 2 小时 / ~3,900 调用 / ~9GB。它通过逐 anchor 重导窗口（而非重放冻结窗口）取代了 §4 提到的更便宜的"window-faithfulness 重打分"半步。

4什么被保证、什么不被保证

直说，便于读者校准信任：

  • 已保证。带泄漏检查的操作点选择（§1）；单一、抽取出的、可追溯的指标内核与 config-match 复用（§2）；对 LLM 致聚合方差的、库存加权的、已测量的界（§3）。
  • Fail-closed，未认证。CT 生产 per-detection 操作点无法在 ≤1pp 迁移下锁定，故以预先声明阈值交付；能认证它的 unified_A × CT 锁已登记但未拟合（tile 待下）。组合阅读者不应推断 CT 操作点已泄漏认证——它是 fail-closed，等待那次拟合。
  • 已记录，非在本文重推。0.97 置信度最优的 ~15pp 计数召回代价，以及 Platt/温度缩放消融，溯源到校准附录自身记录，而非本文一次新运行。
  • 可靠性数字的范围。§3 的冻结输入数字测量的是固定 8 日期窗口下的 JHB install-date 打分器；它界定冻结输入上的采样方差，不含重抓影像或重跑自适应日期搜索带来的额外方差。~81% 是 k=1 生产设置且超采歧义层。早先标记的若干后续现已执行（2026-06-23）：模型一致性重打分确认数字属于生产模型而非 agent 别名；greedy 解码检查显示 temperature:0 已用且不降方差；K=6 majority-of-3 认证在冻结输入上测得去噪提升（→ ~0.95 库存加权）；以及——此前唯一未结的——端到端从 0 重跑（§3"端到端可复现性"）现有 3 次重跑。其裁决重塑此 caveat：冻结输入下界远低于总量。端到端精确区间复现性在两次独立重跑间约 0.74 库存加权（~0.60 vs 生产存量，后者为更高覆盖抽样），伴 46% 层路由 churn——自适应再搜索而非打分器主导总方差，且冻结输入测试结构性地看不见它。幸存的是聚合：install-year cohort 直方图 rep-to-rep TVD 0.037–0.063，故被发表的分布即便个体括号不复现也复现。报告直方图 TVD + rep↔rep 自洽作为复现性证据；把逐装机精确区间作为诊断地板；披露 ~16% 覆盖差作为对生产存量的 caveat。
