### 1. GEID 逆向工程

**动机**：现有标注管线对 allmapsoft GEID GUI 依赖太重，win32 SendMessage 自动化在批量下载时不稳定，且无法做错误重试。Google Maps Tile API 对比测试发现 Z20 API ≈ Z21 GEID，但 GEID 饱和度/锐化更好，最终图源决定继续走 GEID。

**逆向路径**：
- 工具：frida-tools，hook Indy + ssleay32 抓 TLS plaintext
- 域名：`kh.google.com`
- 流程：`dbRoot.v5` → `geauth` (拿 token) → `flatfile?f1-...` 拉瓦片

- 实现：纯 Python 客户端，零 GUI 依赖，commit 到 sibling repo `geid_reverse_engineering/`，PR 到 Eddy 的 GitHub

**自动化优先级（决定）**：WSL 直调 CLI > Python 协议客户端 > SendMessage GUI 兜底。allmapsoft 官方 commandline 模式存在但不能解决失败重试，所以仍以 RE 客户端为主。
