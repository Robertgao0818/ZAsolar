### 晚段 — Bug fix #1: GEHI vintage dedupe 用 version 而不是 capture_date
- 用户指出 GEHI 在 2015-2021 间应有多个 vintage（手工查证 z=21 GEID 之前看得到几个时间节点）
- 诊断：`gehi_common.dedupe_info_rows_by_version` 用 `(anchor_id, version)` 分组，每 version 只留 earliest capture_date。Google Earth `version` 是 stream identifier，跨多 capture_date 复用（v296 在 a000004 横跨 2010-06 到 2021-04），dedupe 把 **74 vintage 砍成 15**
- Fix: 改成 `dedupe_info_rows_by_date`，按 `(anchor_id, capture_date)` 分组，每 date 留一行
- 验证：a000004 z=19 raw 74 → dedupe 后 74，2015-01 到 2022-10 间从 2 个 vintage 变 34 个
- 子仓 commit 3574e67，旧函数名保留为 alias 向后兼容

### 晚段 — Bug fix #2: progressive walk-back 失效（同源）
- 算法本应 R1 (2018-2025) → R2 walk_back (2013-2018) → R3 bisection 这样 5yr/round 渐进
- 实际：R1 (2021-2025) → R2 tail 直接到 2009-2015，跳过 2016-2021 整个区间
- 根因：dedupe 后 in_window (year ≥ 2016) 只有 2 vintage < tail_threshold=3，fallback 到 tail
- Fix #1 自动修复 #2：完整 vintage list 下 in_window ~22 个，walk_back 正常激活
- Trace 验证：用真实 74-vintage list 模拟 a000004 — R1 picks 2018-03 / 2018-12 / 2021-04 / 2022-04 / 2025-05；R2 walk_back 2013-2018 picks 2013/2014/2015/2016/2017；R3 bisection (2017-12, 2018-03) 90 天 high confidence
