## Slide 1. 本周总览：从“检测模型好不好”转向“数据能不能用于经济分析”

### 这周最重要的变化

上周主要还在问：模型有没有漏检、误报多不多、哪个版本 F1 更高。

本周开始把问题往经济学应用推进了一步：

> 我们最终不是只想证明模型能框出太阳能板，而是要得到一个可以按 grid / ward / suburb 聚合、并能和社会经济变量对接的屋顶光伏 panel dataset。

因此，本周工作围绕三件事展开：

1. **聚合口径验证**：用 grid-level 总面积、误差、R² 等指标，看模型结果在统计单元层面是否稳定。
2. **主路线重选**：在 V3-C / V4.1 / V4.2 以及 SAM 后处理之间做同口径比较，选更适合生成“可用安装库”的路线。
3. **误报过滤与普及率验证**：把 classifier 从“热水器过滤器”升级成多类型 non-PV 过滤器，并开始设计 adoption rate（普及率）的建筑底数验证。

### 一句话结论

**本周确认了项目应以“高质量 grid-level solar inventory”为主目标；当前最有希望的技术路线是 V3-C-first + SAM mask refinement + PV/non-PV classifier，而不是继续押单一新 detector。**

## Slide 3. 为什么本周要强调“聚合指标”

### 之前的问题

如果只看单个多边形的 precision / recall / F1，会遇到两个问题：

1. **经济学研究通常不直接用单个 polygon**，而是会按 grid、ward、suburb、census unit 汇总。
2. **单个目标层面的误报和漏检，在聚合层面可能会相互抵消，也可能形成系统性偏差**。所以需要单独测量。

### 本周新增的 DeepSolar-style 聚合指标

04-22 新增 `area_aggregate_eval.py`，把每个 grid 的预测总面积和人工 GT 总面积对比。

| 指标 | 简单解释 | 为什么经济学同事会关心 |
|---|---|---|
| `pred_total_m²` | 模型认为该 grid 内太阳能板总面积是多少 | 可近似转成装机容量或光伏密度 |
| `gt_total_m²` | 人工标注认为该 grid 内真实太阳能板面积是多少 | 作为内部验证基准 |
| `bulk_pred_gt_ratio` | 全部 grid 加总后，模型总量 / GT 总量 | 看是否整体高估或低估 |
| `MRE` | 每个 grid 的平均相对误差 | 看空间单元之间误差是否可接受 |
| `R²` | 模型预测总量与 GT 总量的线性一致性 | 看模型是否能保留空间分布差异 |

### 首轮结果的直观含义

| Region / model run | grid 数 | bulk ratio | MRE/grid | R² | 解释 |
|---|---:|---:|---:|---:|---|
| JHB V3-C | 50 | 0.93 | 0.34 | 0.92 | 总量略低估，但空间排序很稳定 |
| JHB V4 | 50 | 1.07 | 0.35 | 0.89 | 总量接近真实，空间排序也较稳定 |
| CT V3-C | 39 | 1.35 | 0.83 | 0.93 | 空间排序好，但总量明显高估 |
| JHB V4.2 | 10 | 2.19 | 4.79 | 0.44 | 明显过度预测，不适合作为库存数据 |

### 对外解释可以这样说

**如果目标是经济学回归或空间不平等分析，模型不一定需要每个 panel 都完美切对，但必须保证每个空间单元的总量和相对排序可信。本周的结果显示，V3-C / V4 在 JHB 聚合层面已经有可用信号，而 V4.2 会严重高估总量。**

## Slide 4. 本周模型路线如何被重新选择

### 04-24 的初步判断

04-24 复盘后，判断从“继续训练一个更强 detector”转向：

> 用高 recall detector 先尽量找全候选，再用 PV/non-PV classifier 过滤热水器、天窗、阴影、道路标线等误报。

理由是：

- V4.2 的训练 AP50 好看，但真实部署 F1 很差；
- V4-reviewed GT 会偏向 V4，独立 Li GT 显示 V3-C 更稳；
- 误报往往是语义上可识别的 non-PV 物体，适合用小图分类器过滤。

### 04-26 的关键 benchmark

在 25 个 JHB CBD GEID grids 上，用 Li 独立手标 GT，对 V3-C / V4.1 / V4.2 做同口径 benchmark。

| Detector | cluster F1 | matched area F1 | balanced | 直观解释 |
|---|---:|---:|---:|---|
| V3-C | 0.675 | 0.833 | 0.685 | 综合最稳，面积切割好 |
| V4.1 | 0.670 | 0.791 | 0.620 | 没有明显优势 |
| V4.2 | 0.547 | 0.730 | 0.558 | 找得更多，但误报太多、面积质量差 |

### SAM 后处理改变了什么

SAM 可以理解为一个“更会沿边界切图的分割工具”。本周测试了用 detector 给出的 box / mask 去 prompt SAM，让 SAM 重新切太阳能板形状。

完整 9-cell 结果里，最重要的是：

| 组合 | area F1 | balanced | 结论 |
|---|---:|---:|---|
| V3-C + SAM box-only | 0.918 | 0.798 | 面积最准，综合最高，但 recall 有损失 |
| V3-C + SAM mask+box | 0.895 | 0.743 | 更稳健，保留更多检测拓扑 |
| V4.2 + SAM box-only | 0.860 | 0.696 | 面积改善，但 FP 仍太多 |
| V4.2 + SAM mask+box | 0.796 | 0.613 | recall 高，但综合仍不如 V3-C 路线 |

### 本周路线结论

**V3-C-first 重新成为主路线。**

候选主方案：

> V3-C detector → SAM mask refinement → PV/non-PV classifier → grid-level aggregate inventory

这条路线的优点是：

- 原始误报负担比 V4.2 小很多；
- SAM 能显著提高面积切割质量；
- classifier 的任务更现实，不需要从 1200 个 FP 中砍掉绝大多数才能可用；
- 更符合最终要做聚合面积、装机容量、普及率估计的目标。

## Slide 5. Classifier 本周取得了什么进展

### 为什么需要 classifier

Detector 会把一些非太阳能板也报成 PV，例如：

- solar thermal water heater
- skylight / roof window
- corrugated metal roof
- roof shadow
- HVAC equipment
- road marking
- pergola / carport shadow

这些误报如果不处理，会导致 grid-level 装机量被高估，进而影响经济学分析里的 adoption、capacity、neighborhood comparison。

### 04-27 的 subtype audit

从 JHB CBD GEID 的 V3-C / V4.2 共同 FP 池里标注 462 个候选，发现分布如下：

| subtype | 数量 | 含义 |
|---|---:|---|
| actually_pv_mislabeled | 119 | 其实是 Li GT 漏标的真 PV |
| skylight_roof_window | 99 | 天窗 / 屋顶窗 |
| corrugated_metal_roof | 69 | 彩钢屋顶 |
| ground_road_marking | 48 | 地面/道路标线 |
| solar_thermal_water_heater | 32 | 太阳能热水器 |
| pergola_carport_shadow | 30 | 廊架/车棚阴影 |
| roof_shadow_dark_fixture | 25 | 屋顶阴影/暗色设施 |
| HVAC | 21 | 屋顶空调设备 |
| other_unknown | 19 | 其他 |

### 两个重要发现

1. **Cape Town 训练池里的 non-PV 主要是热水器，但 JHB CBD GEID 的误报类型更加多样。**
   - 这解释了为什么 CT-trained classifier 迁移到 JHB CBD GEID 后效果下降。

2. **119 个“误报”其实是 Li GT 漏标的 PV。**
   - 这说明 GT 本身也需要审计；不能把所有 detector 与 GT 不匹配的结果都当成模型错误。

### v1 classifier 的结果

在 16-grid holdout 上，v1 classifier 仍有正收益：

| backbone | precision gain | recall drop | F1 gain | FP removed | TP lost | 解释 |
|---|---:|---:|---:|---:|---:|---|
| EffB0 | +0.101 | -0.036 | +0.036 | 251 | 61 | 轻量 baseline |
| ConvNeXt | +0.110 | -0.029 | +0.044 | 266 | 48 | 当前最稳 default |
| DINOv2 | +0.123 | -0.034 | +0.048 | 292 | 56 | F1 最高但 recall drop 略超预算 |

### v2 protocol 的改进

本周把 classifier v2 从协议推到 **可部署 + 已验证**：

- 从”热水器过滤器”升级为 **multi-subtype FP suppressor**；
- 加入 JHB CBD GEID hard positives 和多类型 hard negatives；
- 按 imagery layer 分开校准 threshold（aerial_2023 / aerial_2025 / geid_2024_02 各一组）；
- promotion rule = **PV recall ≥ 0.95 时 non-PV kill ≥ 0.40**（geid_2024_02 单独门槛）。

### 04-28 完成：v2 三 backbone 训练 + per-layer 校准

| Backbone | aerial_2023 nonpv-kill@R≥0.95 | aerial_2025 | geid_2024_02 |
|---|---:|---:|---:|
| EfficientNet-B0 | 0.50 | 0.75 | 0.56 |
| ConvNeXt-Tiny | 0.65 | 0.79 | 0.51 |
| DINOv2-S/14 | 0.74 | 0.87 | 0.59 |

读数：在保住 95% 真 PV 的前提下，**aerial 层（CT 主流影像）能砍掉 50-87% 的 non-PV，GEID 层只能砍 50-60%。** GEID 上限低主要是 v2 calibration 池里 GEID 真 PV 仅 68 个，pool 仍偏小，而不是分类器没用。

### 04-28 完成：v2 16-grid CT+JHB holdout（per-layer thresholds）

| Backbone | overall F1 | F1 增益 | precision 增益 | recall 损失 | FP 移除 | TP 误删 | 含义 |
|---|---:|---:|---:|---:|---:|---:|---|
| EffB0 | 0.811 | +0.040 | +0.094 | -0.022 | 232 | 37 | 轻量 baseline，性价比可接受 |
| ConvNeXt | 0.819 | +0.047 | +0.105 | -0.019 | 254 | 32 | recall drop 最小，default 候选 |
| DINOv2 | 0.824 | +0.053 | +0.120 | -0.023 | 284 | 38 | F1/precision 最高，激进候选 |

按区域看（DINOv2，最强代表）：

| 区域 | raw F1 | filtered F1 | precision 增益 | recall 损失 |
|---|---:|---:|---:|---:|
| CT (6 grids, aerial 2023/2025) | 0.789 | 0.872 | +0.157 | -0.026 |
| JHB (10 grids, GEID 2024-02) | 0.752 | 0.775 | +0.074 | -0.019 |

**经济学意义**：v2 在 CT aerial 上能让”已检出 PV 中真实 PV 占比”从 68% 提到 84-87%，对 over-counting 风险压降明显；JHB GEID 上 precision 也升到 81-84%，但增益小于 CT，说明 **JHB CBD GEID 仍是当前 grid-level inventory 误差最大的区域，跨域改善是 v3 的核心任务**。所有 backbone 的 recall 损失都控制在 ~2pp 以内，没有出现 v1 时 ConvNeXt -2.9pp / DINOv2 -3.4pp 的过失。

## Slide 6. GT 质量与外部验证：为什么这对经济学研究很关键

### Li GT supplement 的意义

04-27 发现 119 个 detector 高置信预测、人工视觉看是真 PV、但 Li GT 没有标的目标。它们被写成 supplement GT，覆盖 21 个 grids。

这件事的重要性不只是“多了 119 个标注”，而是提醒我们：

> 在南非这种缺少公开 rooftop PV ground truth 的场景下，GT 也不是绝对真理。验证框架必须同时检查模型错误和标注遗漏。

### 对经济学数据产品的影响

如果 GT 漏标很多，直接后果是：

- 模型找到真实 PV 也会被算成 false positive；
- precision 被低估；
- classifier 会被训练去错误地删除真实 PV；
- grid-level adoption / capacity 会被系统性低估或校准错误。

### 本周 V1.4 validation 的方向

本周进一步确认 V1.4 validation 不应只是一张 F1 表，而应包括四类证据：

| Channel | 目的 | 本周进展 |
|---|---|---|
| Stratified precision | 不同区域/影像/目标类型下误报率如何 | classifier subtype audit 已经开始承担这个功能 |
| Exhaustive recall | 小区域内到底漏了多少 | Li GT supplement 暴露了 GT 漏标和模型真阳性 |
| Plausibility | 聚合结果是否符合常识和外部统计 | 04-28 开始 adoption-rate prototype |
| Opportunistic external checks | 能找到的外部数据是否支持模型结果 | 调研 Microsoft / Google building footprints |

## Slide 7. Adoption rate：本周开始接近经济学变量

### 为什么 adoption rate 不是 installed capacity

04-28 明确了一个概念区别：

| 概念 | 需要什么数据 | 典型解释 |
|---|---|---|
| Installed capacity / area | PV 面积或估算功率 | 这个区域装了多少太阳能容量 |
| Adoption rate | 有 PV 的建筑数 / 总建筑数 | 这个区域有多少家庭/建筑采用了太阳能 |

PV 面积可以从 polygon 总面积估计；但 adoption rate 必须有 denominator，也就是建筑/parcel 数量。

### 本周开始做的事

04-28 开始围绕 JHB CBD 25-grid 做 adoption-rate 小实验：

- 查找 Microsoft Global ML Building Footprints、Google Open Buildings 等建筑 footprint 数据；
- 评估它们在 South Africa / Johannesburg 的覆盖和版本；
- 梳理如何把 detection 结果、Li GT / supplement GT、building footprint 按 grid 聚合；
- 把 classifier cascade 的 per-polygon 效果和 grid-level plausibility validation 分开处理。

### 对经济学同事的解释

**如果说装机面积回答“这个地方有多少太阳能”，adoption rate 回答的是“多少建筑/家庭采用了太阳能”。后者更接近 household adoption、能源不平等、收入相关性、政策扩散等经济学问题。**

因此，这一步是在把遥感模型输出转成经济学变量，而不是单纯继续调模型。

## Slide 8. 本周解决的问题与仍存在的风险

### 已解决 / 已推进

1. **评价目标更加清楚**
   - 从 polygon-level F1 转向 grid-level aggregate inventory quality。

2. **主模型路线更清楚**
   - V3-C-first + SAM + classifier 是当前最合理路线；V4.2 降为 control。

3. **SAM 的作用被量化**
   - 它能显著改善面积切割，但 detector box 质量会影响 SAM 上限。

4. **classifier v2 已训练 + 已验证**
   - v1 失败原因（CT-only / water-heater-heavy 训练池）已通过 v2 多子类训练 + per-imagery 阈值得到部分纠正；
   - 16-grid holdout 上 DINOv2 v2 把 overall F1 从 0.771 → 0.824（+0.053），precision +12pp，recall 损失仅 ~2pp；
   - JHB GEID 上仍然是增益最弱的区域（F1 +0.023），需要继续补 GEID hard cases。

5. **GT 质量问题被显性化**
   - 发现 119 个 Li GT 漏标 PV，并生成 supplement GT。

6. **adoption-rate 验证开始启动**
   - 开始引入 building footprint denominator。

### 主要风险

| 风险 | 影响 | 当前应对 |
|---|---|---|
| 南非缺少可靠 external rooftop PV validation | 难以像 DeepSolar 那样直接对接 utility/EIA 数据 | 用 internal GT + aggregate metrics + plausibility checks 组合证据 |
| GT 本身漏标 | 会误判模型 false positive，并污染 classifier | subtype audit + supplement GT |
| classifier 跨域增益不均 | v2 在 CT aerial 上 F1 +0.07-0.08，但 JHB GEID 仅 +0.012-0.023 | per-imagery 阈值已上线缓解；下一步补 GEID hard cases、扩大 GEID 真 PV 校准池（当前仅 68 个） |
| SAM box-only 会损失 recall | 面积更准但漏掉一些对象 | 同时保留 mask+box 稳健路线 |
| adoption rate 需要建筑底数 | 没有 denominator 就不能估普及率 | 调研 Microsoft / Google building footprints |

## 最后给经济学同事的一段简短口径

本周工作的核心不是又训练了一个新模型，而是把项目从”检测算法实验”推进到”经济学可用数据产品”的验证阶段。我们现在更关心的是：在一个 grid 或未来的 ward 里，模型估计的太阳能板总面积、安装数量和采用率是否稳定、是否有系统性高估/低估、是否能和建筑底数及社会经济变量对接。本周的实验证明，V3-C + SAM + classifier 是目前最有希望生成稳定屋顶光伏 inventory 的路线；并且 04-28 完成了 classifier v2 的训练与 16-grid 跨区 holdout 验证——DINOv2 v2 把综合 F1 从 0.771 提到 0.824（precision +12pp、recall 损失仅 ~2pp），其中 Cape Town aerial 的 over-count 风险下降幅度最大。剩下两个关键风险——Li GT 漏标 与 JHB GEID 跨域增益偏弱——是下周首要工作。
