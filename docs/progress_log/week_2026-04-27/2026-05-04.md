### Ch2 area metric 升级（用户反馈“按面积应该更难看”）

- 给 `compute_ch2_recall.py` 加 area-coverage 指标（threshold-free）：`area_coverage_recall = ∑(GT∩pred_union) / ∑GT_area`。
- 发现聚合 area_coverage > count_recall（V3-C 85.9% vs 72.3%）：因为很多 IoU<0.3 的“未匹配 GT”实际有部分 pred overlap，被 area_coverage 收住。
- 但**问题 grid 上大面板真的有问题**：G0925（5 个 xl 中 3 个 0% coverage、l 桶 area_cov 37%）；G0816（三个 ~1320m² 大装机只检测到 4-18%）。grid-specific，不是 model-wide。

### SAM refinement 实验（V3-C / V4.1 × 25 grid Vexcel）

- 修 `scripts/analysis/sam_refine_v4_2_maskprompt.py` 的 `find_chunk_for_geom` CRS bug（Vexcel chunks 是 EPSG:3857，原脚本只支持 4326）。
- V4.1 + SAM(mask+box) on 25 grid：area_R 0.904 → **0.869**（-3.5pp），area_P 0.707 → **0.760**（+5.3pp），IoU 0.658 → **0.682**（+2.4pp），F1 0.794 → **0.811**（+1.7pp），|A|/|B| 1.279 → **1.143**。SAM 修了 over-paint，但还差 V3-C raw。
- V3-C + SAM(mask+box) on 25 grid：area_R 0.859 → **0.817**（-4.2pp），area_P 0.785 → **0.824**（+3.9pp），IoU 0.695 → **0.696**（持平），F1 0.820 → **0.821**（持平），**|A|/|B| 1.094 → 0.992**（接近完美 calibration）。
- 4-way 排名：
  - 最佳 inventory calibration & area_precision: **V3-C+SAM(mask+box)**
  - 最佳 IoU/F1: V3-C+SAM ≈ V3-C raw
  - 最佳 recall: V4.1 raw > V4.1+SAM > V3-C raw > V3-C+SAM
  - V4.1 raw 不可直用（+28% over-paint）
- 结论：**inventory/economic analysis 主路线确定为 V3-C+SAM(mask+box)**；recall-critical / discovery 任务可用 V4.1+SAM 作为互补。
