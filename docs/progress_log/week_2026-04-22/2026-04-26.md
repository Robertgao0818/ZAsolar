### 关键发现

1. **Bounds bug 修复对 cluster 评估几乎无影响**：V3-C rerun (新 chunks) vs 04-19 (旧 mosaic) cluster F1 差 +0.5pp，area F1 差 +0.0001，balanced 差 -0.18pp。5.29% scale 偏差对 cluster matching 不敏感，因为 IoU 阈值有冗余。这说明 04-19 V3-C 数字是可信参考点。
2. **V3-C 在面积切割上显著领先**：matched area F1 0.833 比 V4.1 高 4.2pp，比 V4.2 高 10pp。V4.x 系列通过 HN 训练换来的小目标 recall 没有同步带来更准的 mask boundary。
3. **V4.2 极端 recall 换 precision**：recall 从 V3-C 的 0.729 提到 0.839 (+11pp)，但 FP 从 578 翻到 1211（约 2.1×），cluster P 从 0.628 跌到 0.406。FP 数量爆炸正好对应分类器要解决的 PV vs non-PV 二分类任务。
4. **V4.1 已经被 V3-C 覆盖**：cluster F1 几乎追平 V3-C（0.670 vs 0.675），但 area F1 差 4.2pp，balanced 差 6.5pp，没有任何维度上独立有用。除非作为 ablation control，否则 V4.1 不进入候选。

### V1.4 两个核心问题的回答

- **模型能找出多少太阳能板？** V4.2 (R=0.839) > V3-C (0.729) > V4.1 (0.719)。V4.2 领先最多但带 1211 FP。
- **切出的面积准不准？** V3-C (area F1=0.833) > V4.1 (0.791) > V4.2 (0.730)。V3-C 显著领先，V4.2 切割精度最差。

### 路线决策方向（用户）

用户决定继续以 **V4.2 + PV/non-PV 分类器 + SAM mask head 替换** 为主路线：
- V4.2 的 0.84 recall 是 V3-C/V4.1 都达不到的 ceiling，砍 FP 比补 recall 容易（FP 有结构化模式：水暖、屋顶设施、阴影、路标）
- V4.2 area F1 0.730 偏低反映 Mask R-CNN 的 mask head 表现不佳，**计划换成 SAM** 利用 V4.2 的 box/centroid 作为 prompt 出更准的 mask
- 这条路线与 04-24 "V3-C fixed detector + classifier" 的判断不同：04-24 选 V3-C 是因为它已经给了不错的 area F1；今天的方向反过来，承认 V3-C 的 recall ceiling 才是瓶颈，先解决 recall 再用 SAM 修 mask 质量

## 5. SAM2 Mask Head PoC（同日加做）

用户在看完三档 detector 数字后选择 V4.2 + 分类器 + SAM mask head 三件套路线。下午继续在同 25 CBD grids 上做 SAM2 mask head 替换 PoC，验证 SAM 能否把 V4.2 的 area F1 从 0.730 提到 ≥ V3-C 的 0.833。

### Round 1 — bbox prompt（box only）

脚本：`scripts/analysis/sam_refine_v4_2.py`，使用 V4.2 polygon 的 bbox 作 SAM2 `input_boxes` prompt，`multimask_output=True`，取 `argmax(score)` 的 mask。

实现踩坑：
- 当前 transformers Sam2Processor `input_boxes` 是 3 级嵌套（不是文档常见的 4 级）
- `outputs.pred_masks` 是 bfloat16，转 numpy 必须先 `.to(torch.float32)`
- 也不再有 `reshaped_input_sizes` key，`post_process_masks` 签名改成 `(masks, original_sizes, ...)`

A4500 + bfloat16 + sam2.1-hiera-large，2170 prompts in 5 min（~7.2 prompts/s）。

| | matched | FP | FN | cluster R | cluster F1 | matched area F1 | cluster IoU | balanced |
|---|---:|---:|---:|---:|---:|---:|---:|---:|
| V4.2 (no SAM) | 826 | 1211 | 159 | 0.839 | 0.547 | 0.730 | 0.584 | 0.558 |
| V4.2 + SAM (box) | 821 | 1211 | 244 | 0.771 | 0.530 | **0.860** | **0.776** | **0.696** |

Area F1 +12.9pp、cluster IoU +19.2pp、balanced +13.8pp（**0.696 全场最高**，超 V3-C 0.685）。**但 cluster F1 反而 -1.6pp**，FN 从 159 涨到 244（+85）。

### Round 2 — Union(SAM, V4.2) 救 recall（用户提议）

用户假设：是不是 SAM 切小了，cov_gt 跌破 0.5 阈值导致 recall 掉？所以做 `union(SAM, V4.2)`：

| | matched | FP | FN | cluster R | cluster F1 | matched area F1 | balanced |
|---|---:|---:|---:|---:|---:|---:|---:|
| V4.2 + SAM-union | 794 | 1231 | 177 | 0.818 | 0.530 | 0.721 | 0.560 |

**Union 失败**：area F1 反掉到 0.721（比纯 V4.2 还差）+ matched 跌到 794。原因：
- 90.7% SAM ⊂ V4.2，union ≈ V4.2 → area F1 应回到 V4.2 水平
- 9.3% SAM 切到 V4.2 mask 外面（SAM 切偏到邻近物体），union 把这些"脏边"加进来
- union 后相邻 pred 通过细线连成超大连通体，cluster_eval 多个 1-vs-1 match 合并成 1 个 N-vs-M cluster，触发 cardinality penalty

### Round 3 — FN 失败模式诊断

用 cluster_details.json 比对 V4.2 vs V4.2+SAM(box)，找出 170 个 GT 个体在 V4.2 matched 但 SAM FN：

| Mode | n | 比例 | 现象 |
|---|---:|---:|---|
| **A. SAM mask 移位/选错** | 104 | 58% | sam_iou_with_gt = 0（SAM mask 不再碰这个 GT） |
| **B. Cluster matching 重组** | 74 | 42% | pred ↔ GT 配对仍 OK，但 connected component 重新分组失配 |

Mode A 数据：V4.2 大 mask（中位 211m²）覆盖小 GT（中位 26m²）→ V4.2 实际是"过度分割，一个大框包多个小 panel"。SAM 用 bbox prompt 在这个 211m² 矩形里切，`argmax(score)` 选了一个 108m² **sub-component**（specificity bias），落在 V4.2 大框里**另一个 panel** 上，完全不碰这个 GT。

**结论**：FN 涨不是面积变小，是 **SAM 选错 mask**。bbox prompt 信息量不够——丢了 V4.2 polygon 的形状本身。

### Round 4 — Mask-prompt mode（mask + box）

脚本：`scripts/analysis/sam_refine_v4_2_maskprompt.py`，把 V4.2 polygon 栅格化成 256×256 binary，转成 logits（±10）作 SAM2 `input_masks`，叠加 box anchor。HuggingFace transformers 的 `Sam2Processor` `__call__` 不支持 `input_masks` 参数，但 `Sam2Model.forward` 直接接收，绕过 processor 手动加。

| | matched | FP | FN | cluster R | cluster F1 | matched area F1 | balanced |
|---|---:|---:|---:|---:|---:|---:|---:|
| V3-C | 977 | 578 | 364 | 0.729 | **0.675** | 0.833 | **0.685** |
| V4.2 | 826 | 1211 | 159 | 0.839 | 0.547 | 0.730 | 0.558 |
| V4.2 + SAM (box) | 821 | 1211 | 244 | 0.771 | 0.530 | **0.860** | 0.696 |
| **V4.2 + SAM (mask+box)** | **841** | 1198 | **155** | **0.844** | 0.554 | 0.796 | 0.613 |

**FN 完全恢复**：155，甚至比 V4.2 的 159 还少 4 个；recall 0.844 全场最高。**但 cluster F1 仍仅 0.554** —— 因为 FP 完全没动（1198 ≈ 1211）。

Mask 收缩程度大幅缓解：
| | box-only | mask+box |
|---|---:|---:|
| sam/orig 中位 | 0.723 | 0.858 |
| sam < 0.5× | 8.0% | 0.1% |
| sam < 0.25× | 1.4% | 0.0% |

### 关键判断

1. **Mask head 替换可行性已验证**：mask-prompt 模式在保 detection 拓扑（FN 不增）的同时把 area F1 从 0.730 → 0.796（+6.6pp），仍低于 box-only 的 0.860 但高于 V4.2 / V4.1 / V3-C-baseline。
2. **真正瓶颈不是 mask head 是 FP**：mask-prompt 已把 recall 0.844 + area F1 0.796 推到 detection 端极限，cluster F1 仍卡在 0.554，因为 1198 FP 把 cluster precision 锁在 0.41。**剩下提升必须靠分类器**。
3. **Box-only vs mask-prompt 的取舍**：
   - box-only：area F1 0.860（最准切割），但 -85 个 GT 漏（FN 从 V4.2 的 159 → 244）
   - mask+box：area F1 0.796（比 V3-C 略低），但 FN 155（比 V4.2 还稳）
   - 实战选 mask+box——先保 detection 拓扑稳定，area gain 留给后续 SAM prompt 调优
4. **三件套预估**（V4.2 + mask-prompt SAM + 分类器砍 70% FP）：
   - FP: 1198 → ~360
   - cluster P: 0.413 → ~0.70
   - cluster R: 0.844（不变）
   - cluster F1: ~0.77（V3-C 0.675 +9.5pp）
   - area F1: 0.796
   - **balanced: ~0.77（全场最高）**

> **注**：以上 V4.2-route PoC 的判断在晚间被 V3-C / V4.1 SAM ablation 覆盖。完整 9 档对照见 section 7，V3-C + SAM 路线起点高于 V4.2 + SAM。

## 6. 完整 3×3 SAM ablation（晚间，本地 RTX 4070）

下午看完 V4.2 PoC 后用户原本选 V4.2 + SAM mask+box 三件套；但脚本内部存在隐藏假设 ——"V4.2 是 over-seg detector，所以 box-only 的 FN flip 是 V4.2-specific，换 V3-C/V4.1 不会有"，以及"SAM mask 是 area F1 上限，detector 选型只看 recall + topology"。pod 已关，本地 venv 修好后在 RTX 4070 8GB（bfloat16 + sam2.1-hiera-large）跑了 V3-C 和 V4.1 各两档（mask+box / box-only），共 4 个 SAM refine + 4 个 cluster_eval，单次推理 ~12-13 min（与 A4500 持平）。结果证伪两个假设。

### 完整 9 档矩阵

| Detector / SAM mode | matched | FP | FN | cluster R | cluster F1 | **area F1** | **balanced** |
|---|---:|---:|---:|---:|---:|---:|---:|
| V3-C / no SAM | 977 | 578 | 364 | 0.729 | **0.675** | 0.833 | 0.685 |
| **V3-C / box-only** | 948 | 591 | 495 | 0.657 | 0.636 | **0.918** ⭐ | **0.798** ⭐ |
| V3-C / mask+box | 981 | 578 | 389 | 0.716 | 0.670 | 0.895 | 0.743 |
| V4.1 / no SAM | 851 | 508 | 332 | 0.719 | 0.670 | 0.791 | 0.620 |
| V4.1 / box-only | 831 | 519 | 467 | 0.640 | 0.628 | 0.891 | 0.750 |
| V4.1 / mask+box | 857 | 506 | 339 | 0.717 | 0.670 | 0.855 | 0.676 |
| V4.2 / no SAM | 826 | 1211 | 159 | **0.839** | 0.547 | 0.730 | 0.558 |
| V4.2 / box-only | 821 | 1211 | 244 | 0.771 | 0.530 | 0.860 | 0.696 |
| V4.2 / mask+box | 841 | 1198 | **155** | **0.844** | 0.554 | 0.796 | 0.613 |

### 三个关键发现

1. **"SAM 是 area F1 上限" 假设证伪**：V4.2+SAM 顶 0.860，V3-C+SAM 顶 0.918，**detector box 质量直接传递到 SAM 输出**。原假设以为 SAM 会把所有 detector 的 mask 都拉到同一个 ~0.79-0.80 收敛点，让 detector 选型只看 recall + topology；事实是 detector 的 box 形状 / boundary 对 SAM 选 sub-component 起到强先验作用，清洁 box 让 SAM 切得更准。
2. **box-only FN flip 不是 V4.2-specific**：V3-C +131 FN，V4.1 +135 FN，V4.2 +85 FN — 三家 detector 都掉 recall。但 V3-C 因 baseline matched 977 大、FP 不变、area F1 暴涨，balanced 反涨到 0.798（前高 V4.2+SAM-box 0.696，**+10.2pp**）。即"SAM `argmax(score)` specificity bias 系统性偏向 sub-component"是普遍模式，不是 over-seg detector 独有。
3. **mask+box +6pp area F1 是 detector-invariant 增益**：V3-C +6.2 / V4.1 +6.4 / V4.2 +6.6。SAM mask-prompt 的核心价值是补充 polygon 形状信息抑制 sub-component flip，不是修 V4.2 over-seg。

### 路线判断修正

- **V3-C 改回主路线**：新候选 = **V3-C + SAM mask+box + classifier**（保 recall 0.716 + area F1 0.895 + FP 仅 578）。
- 三件套预估（V3-C + SAM mask+box + classifier 砍 70% FP）：
  - FP: 578 → ~173
  - cluster P: 0.629 → ~0.85
  - cluster R: 0.716（不变）
  - cluster F1: ~0.78（V3-C baseline 0.675 +10.5pp，估算高于 V4.2-route 的 ~0.77）
  - area F1: 0.895
  - **balanced: ~0.80+（估算全场最高）**
- **激进备选 = V3-C + SAM box-only + classifier**：起点 area F1 0.918 / balanced 0.798，分类器砍 70% FP 后 cluster F1 ~0.74、balanced 估算 ~0.83+，但 recall 0.657 是天花板。如果终极 KPI 偏 area precision、recall 让位，这条更优。
- **V4.2 路线降为 control**：保留作为"高 recall + 大量 FP"极端控制点，仅当分类器极强（>90% FP recall）时再考虑切回 V4.2 起点。
- **V4.1 出局确认**：每一 SAM mode 下都被 V3-C 同 mode 覆盖。
