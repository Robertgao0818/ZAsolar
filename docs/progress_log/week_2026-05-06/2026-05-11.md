### 3) Codex review fix（3 个 finding 全采纳）

**P1 #1 build_unified_reviewall.py:369**：`balance_chips(all_images, args.neg_ratio, seed=args.seed)` 签名错。真签名是 `(images, annotations, provenance, seed, neg_ratio)`，默认 `--neg-ratio 0.15` 下非 dry-run 会 TypeError。修：`balance_chips(images, annots, prov, seed=..., neg_ratio=...)` + 去掉手动 filter annots 那几行。

**P1 #2 mask_pixel_weights 空间错位**：`maskrcnn.py` 没传 `min_size/max_size`，torchvision 默认会把 400 chip resize 到 800；`target['masks']` 跟着 resize 但 custom dict 字段（`mask_pixel_weights`, `ignore_masks`）不跟。pre-forward hook 在 transform 之前 stash 400-space 权重，patched_maskrcnn_loss 用 800-space proposals 去 ROIAlign-project → 2× 错位采样。

修：加 `install_transform_aux_resize(model)` wrapping `model.transform.forward`，在 base transform 跑完后用 `F.interpolate` 把 `mask_pixel_weights` (bilinear) 和 `ignore_masks` (nearest) resize 到 post-transform 尺寸再 stash。train.py 把 `register_forward_pre_hook` 改成 `install_transform_aux_resize(model)`。box-loss bucketing 路径同样切到 transform wrap（label_sources 非 spatial，无影响但语义统一）。

端到端单测：400×400 chip → 800×800 post-transform，aux 字段同步 resize，bilinear 后 mean 0.7 保持，stash 形状对齐。

**P2 mask_trusted_for(None) 默认 True**：legacy CT/JHB reviewed/V4 export 路径若 GDF 缺 `label_source` 列会把所有 ann 默认标 `mask_trusted=True`。unified builder 自己补了 label_source 没中招，但通用 export 路径不完整。

修：
- `mask_trusted_for(None)` raise + 未知 source 也 raise（强制显式映射）
- `scan_chips_from_tile` 把 `ann["mask_trusted"]` 赋值移进 `if label_source is not None` 块内 → legacy 路径不再写
- `train._resolve_mask_trusted` 兜底：未知 source → False（保守 untrusted）；都缺 → True（legacy 兼容）

3 个 fix 通过 py_compile + 单测：`balance_chips` 3-arg + neg_ratio kwarg 工作；`mask_trusted_for(None)` 和 unknown source 都 raise；transform wrap 端到端 resize 正确。

## 决策与判断

1. **train.py 大 commit 合并**：原计划按 plan 切 5-6 个 sub-commit（CocoSolarDataset / freeze-mask / 差分 LR / box-log / early stop），但 hunk 跨度太大、`git add -p` 不便。改成单 commit + comprehensive message 列举所有 sub-feature。工程纪律 "每条独立 commit" 在 boundary_aware_mask.py / export / build / yaml 四个独立文件上守得很干净，train.py 一条退让。

2. **codex fix 单独 commit 而非合进 feature**：fix commit 必须独立的工程纪律本可以放松（fix 是 review 后 same-day 即修），但保留分离让 git log 可追溯 "feature 上线时是 buggy 的，codex 在 commit 后 4 小时内 catch + fix"。Review 流程的存在感不能在 history 里抹掉。

3. **PR Codex 的 OR-stop 语义 caveat**：codex 提示 OR-stop 在 "AP50 不涨但 F1 还在涨" 时会过早停止，这是 plan 本来就要的语义（防 collapse），不修。但记在 commit message 里以备未来 reviewer 看到时知道是 intentional。

4. **clean_baseline_hs 的 commit 顺序**：clean_baseline_hs 在 yaml 里早写了几行；unified_reviewall 是今天写的；二者都未 commit。本来应该 clean_baseline_hs 先 commit，但因为这次想保住 unified_reviewall 的工程纪律完整（mask_trusted 系列的 yaml entry 独立 commit），所以 split yaml 时先 commit unified_reviewall entry，再 restore + commit clean_baseline_hs 入它自己的 builder commit。git history 顺序不影响功能，但 timeline 含义稍微不直观。

5. **Auto-mode 边界**：codex review 提的三条 fix 全部低风险 + local + 有单测覆盖，符合 auto-mode 的 "make reasonable assumptions and proceed on low-risk work"。最后清场的 9 个 topical commit 也是同样低风险（都是把已落盘的工作 commit 化）。整个 day 没有触及 destructive / shared / production 边界，无需 user 确认。
