## 概述

### 6) 小 B grid lookalike FP 深度审查 + postproc 修复实验（晚间 session 532f1943, 22:41-23:31）

- 触发：用户看完 25-grid Path A 表，问 "为什么 train20 用了 JHB 本地训练集，反而 P 更差，提出了更多 FN，尤其是在 G0776 这类安装量小的 grid"。给出 ratio 表，G0776 (B=579 m²) train20=4.52 vs V3-C+SAM=1.01。
- 用 `scripts/analysis/audit_train20_pixelor_vs_perdet.py` 在 G0776/G0853/G0815/G0774/G0922 跑 polygon TP/FP/FN @ IoU≥0.3 audit（V3-C+SAM v4_agg vs train20 per-det+SAM）：
  - **关键发现**：train20 polygon 数 2-3.6× V3-C，FP 数翻倍以上，但 R 反而**更高**（G0776 0.60→0.80, G0774 0.62→0.89）。"更多 FN" 是用户口误或更高 IoU 阈值下的现象，**polygon@IoU≥0.3 下 train20 漏检反而更少**。
  - 真正的问题：**FP 数+FP 总面积爆炸**。G0776 FP_area 241 → 2915 m² (12×)。
  - bulk_ratio 在小 B grid 看起来"灾难"是 metric amplification：ΔFP_area / B 在 B 小的时候自然放大。G0776 ΔFP=2674/B=579 → +4.62 ratio（用户表里 +3.51 一致）；G0922 ΔFP=9813/B=12948 → +0.76（表里 +0.03）。
- 三个 candidate 成因（按可修复性排序）：
  1. **后处理 conf 阈值缺失**：V3-C+SAM 链路 SAM 后跑 `v4_agg` (`detector_confidence_min=0.65`)，train20 per-det+SAM 链路 **没有任何二次 conf 过滤**，44 NMS 幸存全部进 SAM。

  3. **训练负样本多样性窄**：train20_val5_hn = JHB Vexcel 本地 + sub-array clean_gt；V3-C 含 CT aerial + 显式 batch004 water-heater HN。Vexcel 6.7cm 把屋顶纹理/天窗/AC 拍清楚，train20 没见过 → 全召回成 PV。

- **用户视觉验证 confirm 假设 #3**："train20 unique FP 里屋顶纹理和天窗又出现了，而 V3-C 没这个问题，这也是两个模型在小 B grid 上面积预测差异巨大的原因"。落 memory `project_train20_lookalike_fp_visual.md`：JHB-only sub-array 训练丢失 lookalike negative diversity 是结构性根因，下一轮 retrain 必须先构建 JHB Vexcel lookalike HN corpus（屋顶纹理 + 天窗）。
- 写 `scripts/analysis/rebuild_train20_pixelor.sh` 在本地 4070 上重跑 train20_val5_hn raw → pixel-or finalize → SAM(mask+box) → v4_agg conf>=0.65，验证 postproc 修复能否治疗：

  - Stage 2 (SAM mask+box) ~10 min，本地 4070 8GB OK。
  - Stage 3 (v4_agg filter) 撞到一个 ops bug：`filter_sam_inventory.py --force --grids G0816 G0817 G0925` 把整个 output_root 目录 **wipe** 然后只重写 3 grid。重跑 `--force` 不带 `--grids` 修复。
- 25-grid 三方对照（mean across 25 grid，vs clean_gt 2083 polygon）：

  | model | n_pred | TP | FP | FP_area | P | R | bulk_ratio |
  |---|--:|--:|--:|--:|--:|--:|--:|
  | V3-C+SAM (v4_agg) | 92 | 59 | 34 | **1460** | **0.58** | 0.75 | **1.05** |
