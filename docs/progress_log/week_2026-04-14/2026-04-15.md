### 3. G1110-G1114 算法表现分析（中途两次迭代）

**定义**（用户指定语义）:
- `correct + edit` = TP（edit = 识别对，只是边界切错）
- `delete` = FP
- `sam_fn_review` 与 `edit` 多边形空间重叠 → 视为 Edit 的边界修正（不计入 recall，避免重复计数）
- `sam_fn_review` 不重叠 edit → 真 FN

**第一次跑**（G1110/G1111 未做 FN pass）:
- G1110/G1111 recall=100%（不可信，SAM-FN=0）
- TOTAL: P=0.851 / R=0.712 / F1=0.775

**补做 G1110/G1111 FN pass 后**（G1112 也额外补了 +77 FN marker）:

|------|------:|---:|-----:|------:|------:|------:|

**对比 CBD batch1 V4**（P=0.684 / R=0.827 / F1=0.748）:

| 指标 | Sandton 5-grid | CBD batch1 | Δ |
|------|---------------:|-----------:|--:|
| P    | 0.851 | 0.684 | **+16.7pp** |
| R    | 0.617 | 0.827 | **-21.0pp** |
| F1   | 0.715 | 0.748 | -3.3pp |

**核心信号**:
- Precision 比 CBD 好（FP 变少），但 **recall 崩 21pp** → Sandton 主要瓶颈在漏检
- Per-grid recall 两极分化：稀疏 grid (G1110=40preds/G1111=149preds) R=77-89% 正常；密集 grid (G1112=137/G1113=122) R=51-54% 几乎漏一半
- Edit 修正多边形占 SAM-FN 的 10/298=3.4%（5-grid 合计），全部落在 Edit 原多边形内，和用户"修正多边形落在 Edit 内"观察一致

**可能原因**（按优先级）:
1. **2023 航测图 vs GEID 影像 domain gap** —— V4 在 GEID CBD 影像上训练，现在推的是 2023 航测，分辨率/拍摄角度/季节都不同
2. **Sandton 屋顶类型与 CBD 不同** —— residential 为主，装置更小、更密、更多种类型
3. V4 Joburg 训练集全来自 CBD (G0772-G0926)，Sandton 有 domain shift
